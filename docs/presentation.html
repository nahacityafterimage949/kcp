O PROBLEMA

Conhecimento gerado por IA está se perdendo

Toda empresa que usa IA hoje vive o mesmo ciclo: gerar → perder → regerar → perder. Nenhuma camada do stack de rede atual resolve isso.

💭
Conhecimento Efêmero
Análises ficam em chats privados, desaparecem quando a sessão termina. Zero reuso entre times.
🔍
Descoberta Impossível
"Alguém já analisou isso?" → ninguém sabe onde procurar. Cada time é uma ilha isolada.
🔒
Governança Zero
Sem rastreabilidade de fontes, sem versionamento, sem lineage. Impossível auditar decisões.
🏝️
IAs em Silos
ChatGPT não sabe o que o Claude fez. Copilot não sabe o que seu agente custom gerou. Zero interoperabilidade.
PRIOR ART

Por que nada que existe hoje resolve?

Pesquisamos extensivamente: Semantic Web, MCP, Knowledge Graphs, Data Catalogs, Git, Wikis, RAG. Nenhum combina persistência + governança + descoberta + lineage em um protocolo aberto.

🌐 Semantic Web (RDF/SPARQL)
W3C, 2001 — 25 anos
✓ O que faz bem:
Dados linkados estruturados, ontologias formais (OWL)
✗ O que falta:
Foco em dados estáticos. Sem lineage. Sem multi-tenant. Adoção <0.1% da web.
🔌 Model Context Protocol (MCP)
Anthropic, 2024 — 2 anos
✓ O que faz bem:
Compartilha contexto entre ferramentas IA em tempo real
✗ O que falta:
Apenas sessão. Sem persistência. Sem descoberta. Sem governança.
🕸️ Knowledge Graphs
Google KG, Wikidata, Neo4j
✓ O que faz bem:
Relações entre entidades, graph traversal
✗ O que falta:
Modela entidades, não insights. Sem lineage. Sem multi-tenant. Curadoria manual.
📂 Git / Wikis / RAG
Diversas tecnologias
✓ O que faz bem:
Versionamento (Git), documentação (Wiki), retrieval (RAG)
✗ O que falta:
Sem busca semântica. Sem auto-ingest de IA. Sem federação cross-org. Fragmentados.

PROTOCOLO

KCP: Knowledge Context Protocol

Um protocolo aberto para interoperabilidade de conhecimento entre IAs, ferramentas e organizações.

Auto-Ingest
📝
Publicação Sem Fricção
Agentes publicam outputs automaticamente. HTML, JSON, análises — tudo armazenado com metadata rico.
Lineage
🔗
Rastreabilidade Total
Cada artefato registra: fonte de dados → query → agente → insight → decisão. Cadeia completa e auditável.
Discovery
🔍
Busca Semântica
Vector DB + full-text search. "Quem já fez análise de churn?" retorna resultados relevantes em milissegundos.
P2P
🌐
Federação Distribuída
Inspirado em BitTorrent/IPFS. Cada organização é um nó. Sem servidor central. DHT para descoberta.

SEGURANÇA

Zero-trust por design

Cada artefato é assinado. Cada acesso é verificado. Cada tenant é isolado criptograficamente.

🔐
3 Camadas de Criptografia
Transporte: TLS 1.3
Storage: AES-256-GCM (at rest)
Conteúdo: Per-tenant keys (E2E)
✍️
Assinatura Digital
Ed25519 obrigatório. Todo artefato é assinado pelo autor. Non-repudiable. Verificável por qualquer nó.
🏢
Isolamento Zero-Knowledge
Dados do Tenant A são criptografados com a chave do Tenant A. Tenant B não pode ler, mesmo com acesso físico ao storage.
🎚️
4 Tiers de Visibilidade
Public: Qualquer um lê
Org: Mesmo tenant
Team: Mesmo time
Private: Só autor + ACL

ENTERPRISE

Três caminhos para produção

Do uso pessoal ao enterprise — escala sem reescrever código.

📁

Google Drive / Dropbox

Pasta compartilhada como transport. SDK escreve artifacts na pasta, cloud sync distribui pra todos. Zero infra.

KCP_SHARED="/My Drive/kcp"
🏢

Hub Corporativo (EC2)

FastAPI + PostgreSQL + Nginx. Auth via API Key ou OAuth2. Multi-tenant com ACL e audit log.

EC2 t3.small = ~$17/mês
🌐

Federation

Hubs se conectam entre si. Empresa A ↔ Empresa B compartilham knowledge com mTLS + ACL.
